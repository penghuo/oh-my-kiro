# Team MVP Implementation Plan

> **For Kiro:** Use `#executing-plans` for sequential tasks.

**Goal:** End-to-end MVP: user invokes team skill in kiro-cli → spawns a code review worker via ACP in tmux → orchestrator consolidates result.

**Architecture:** A kiro-cli skill (`team`) tells the LLM to call a Node.js script (`spike/team-mvp.mjs`). The script generates a worker agent config, spawns `kiro-cli acp` as a child process, creates a tmux pane showing live progress, sends the review task via ACP JSON-RPC, and returns the result to stdout. The orchestrator LLM reads the result and presents it.

**Tech Stack:** Node.js (ESM), kiro-cli ACP (JSON-RPC over stdio), tmux

### Task 1: ACP Client Library

Extract the proven ACP client from `spike/acp-spike.mjs` into a reusable module.

**Files:**
- Create: `spike/lib/acp-client.mjs`

**Step 1: Create the ACP client module**

```javascript
// spike/lib/acp-client.mjs
// Exports: createAcpClient(name, opts) → { initialize, newSession, prompt, kill, onChunk }
// Wraps: spawn kiro-cli acp, JSON-RPC send/receive, chunk streaming
```

Key functions:
- `createAcpClient(name, { agent, trustAllTools })` — spawns `kiro-cli acp` child process
- `client.initialize()` — sends `initialize` JSON-RPC, returns capabilities
- `client.newSession(cwd)` — sends `session/new`, returns sessionId
- `client.prompt(sessionId, text)` — sends `session/prompt`, streams chunks via callback, returns full response text when `stopReason: "end_turn"`
- `client.kill()` — terminates child process
- `client.onChunk(callback)` — register callback for streaming text chunks

**Step 2: Verify it works**

Run: `node -e "import('./spike/lib/acp-client.mjs').then(m => console.log('ok'))"`
Expected: `ok`

**Step 3: Commit**

```bash

git commit -m "feat: extract ACP client library from spike"
```

### Task 2: Team MVP Script

The main orchestrator script that generates agent config, spawns worker in tmux, and collects results.

**Files:**
- Create: `spike/team-mvp.mjs`

**Step 1: Write the team MVP script**

The script does:
1. Parse args: `--role <role>` `--task <task>` `--context-file <path>` (or `--context-cmd <cmd>`)
2. Generate worker agent config JSON to `/tmp/omk-worker-<id>.json`
3. Create tmux split-pane running `tail -f /tmp/omk-worker-<id>.log`
4. Spawn `kiro-cli acp --agent <config> --trust-all-tools` as child process
5. Initialize → newSession → prompt with the review task
6. Stream chunks to log file (visible in tmux pane)
7. On completion, print final result to stdout
8. Clean up tmux pane

Agent config template:
```json
{
  "name": "omk-review-<role>",
  "description": "Code reviewer focusing on <role>",
  "prompt": "You are a code reviewer. Focus on <ROLE>. Review the code diff provided. Give specific, actionable findings with file and line references. Be concise.",
  "tools": ["read", "shell", "grep", "glob"],
  "allowedTools": ["read", "shell", "grep", "glob"],
  "model": null
}
```

**Step 2: Test manually**

```bash
# Create a test diff
git diff HEAD~1 > /tmp/test-diff.txt

# Run the script
node spike/team-mvp.mjs \
  --role "maintenance" \
  --task "Review this PR for maintenance issues" \
  --context-file /tmp/test-diff.txt
```

Expected: tmux pane shows streaming progress, stdout shows final review result.

**Step 3: Commit**

```bash
git add spike/team-mvp.mjs
git commit -m "feat: team MVP script with ACP + tmux"
```

### Task 3: Team Skill

The kiro-cli skill that teaches the orchestrator LLM how to use the team script.

**Files:**
- Create: `~/.kiro/skills/team/SKILL.md`

**Step 1: Write the skill**

```markdown
---
name: team
description: "Spawn agent teams for parallel work. Use when user says 'team' or 'teams' followed by a task like code review."
inclusion: manual
---

# Team Orchestration

## Overview
Spawn specialized kiro-cli worker agents to perform tasks in parallel.
Workers run in tmux panes for visibility. Results are consolidated by you.

## When to Use
User says "team" or "teams" followed by a task description.
Example: "teams, spawn a code reviewer to review PR for security"

## How to Use

### Step 1: Prepare context
Get the diff or relevant code:
- For PR review: `git diff main..HEAD > /tmp/omk-review-context.txt`
- For CR review: fetch the CR diff

### Step 2: Launch worker
Run via execute_bash:
node <path>/spike/team-mvp.mjs \
  --role "<aspect>" \
  --task "<specific task>" \
  --context-file /tmp/omk-review-context.txt

The script:
- Spawns a kiro-cli worker in a tmux pane (visible to user)
- Sends the task via ACP
- Returns the worker's review when done

### Step 3: Present results
Read the output and present a consolidated summary to the user.
```

